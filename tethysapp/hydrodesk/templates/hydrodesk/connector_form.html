A local .nc path or a remote OPeNDAP/dodsC URL. {field} placeholders are filled from the record.
Comma-separate several variables (e.g. SST, AIRT, SLP) to get a combined table plus a per-variable series + latest value. Click Test below to list the dataset's variables. Each is averaged over its non-time dimensions. Declare Inputs below to substitute {field} into the dataset/catalog URL.
One name = formula per line. The formula is evaluated per row over the real variables (by their exact names) and added as an extra table column. Operators + - * / ** %; functions sqrt, abs, min, max, round, floor, ceil, log, exp.
{# ---- Spatial filter (how the lat/lon axes are reduced) ---- #}
Whole-grid = average every cell. Point = the cell nearest a lon/lat (leave blank to use the record's geometry). Bbox = average inside a lat/lon window (leave the bounds blank to centre it on the record ± buffer). Polygon = average the cells inside a shapefile/WKT region. Per-cell = NO averaging — each grid cell in the box is its own column. Per-zone = import a MULTI-polygon shapefile (a grid/cells/subbasins layer) and get the mean inside EACH polygon as its own column.
Upload a zipped shapefile (.shp + .shx + .dbf) or a bare .shp. On save it is converted to inline GeoJSON — no file is kept on the server. In Polygon mode the first polygon becomes the region; in Per-zone mode EVERY polygon is imported (one column each). (Then re-open to Test, which reads the stored geometry.)
In Polygon mode the grid cells inside the first polygon (holes excluded) are averaged; in Per-zone mode every polygon in the file becomes its own column. Lon is normalised to −180…180; a polygon crossing the antimeridian isn't supported yet.
Per-cell uses the Bounding box bounds above to pick the region (or the record ± buffer when blank). Each cell becomes a column labelled lat,lon; if the box has more than this many cells, evenly-spaced ones are sampled.
Which shapefile attribute names each zone's column. Blank = a well-known id field (name/id/huc/gridid…) or the polygon centroid.
Caps how many polygons become columns (extras dropped).
Per-zone imports a multi-polygon shapefile (above) — a grid, cells, or subbasins layer. Each polygon → one column = the variable's mean over the cells inside it. (You can also paste a GeoJSON FeatureCollection/MultiPolygon in the Polygon box.)
{# ---- CSV source ---- #}
A remote http(s) CSV or a local file path. {field} placeholders are filled from the record. The whole table becomes one series (each column a variable).
Click Test below to preview the columns. The "latest" value comes from the Value column's last non-empty row.
{# ---- WMS source ---- #}
The base WMS endpoint. A GetMap image centred on the record's point (longitude/latitude) is shown on the detail. Click Test to list the service's layers.
The map centres on the record's geometry (a point type) or its longitude/latitude fields. The default centre is the fallback for non-spatial doctypes. A best-effort GetFeatureInfo value at that point is also available as a second output.
{# ---- WCS source ---- #}
A WCS 2.0.1 endpoint. The coverage value near the record's point is read by requesting a small NetCDF subset. Click Test to list coverage IDs.
The point comes from the record geometry / longitude-latitude fields (default centre for non-spatial types). The primary output is the coverage's spatial-mean value near the point; add a time range in Extra subset to get a series.
{# ---- Google Earth Engine source ---- #}
An ee.Image id for a value, or an ee.ImageCollection id (with a date range below) for a series. Requires the earthengine-api package + a service-account credential.
A stored credential whose secret is the service-account key JSON. Add one under Credentials. Click Test to confirm Earth Engine initialises.
