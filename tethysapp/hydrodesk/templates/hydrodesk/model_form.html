Execution
When on, creating or editing a record of a doctype bound to this model fires the run immediately on save — the outputs populate on the record page on their own (it polls and refreshes when the run finishes), with no manual Run click. Leave off for heavy/expensive models you'd rather trigger by hand.
Default runs on a local worker thread. HTCondor/Dask submit to a Tethys scheduler of the given name — if that scheduler isn't configured on this portal, the run falls back to the thread (the job log records it). Long runs (>1 h) pin a thread — prefer a cluster backend for those. Track every run on Model Jobs.
